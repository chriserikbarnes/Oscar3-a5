Name Recognition DFAs

This document describes the Discrete Finite Automata that are used to recognise named entities. The role of these systems is described in the name recognition help file.

Data files

There are three data files for these modules: uk/ac/cam/ch/wwmm/oscar3/terms/resources/neTerms.txt uk/ac/cam/ch/wwmm/oscar3/terms/resources/custEnt.txt, and uk/ac/cam/ch/wwmm/oscar3/terms/resources/ontology.txt. In addition, multi-token entities from the ChemNameDict are also used. Of these, neTerms.txt and the multi-token entities from the ChemNameDict are only used when the MEMM is not in use, whereas ontology.txt and custEnt.txt are used regardless of whether

The data files consist of a number of sections, denoted using headings in square brackets, like [CM]. In neTerms.txt these sections map onto what type the named entity is to be, whereas in ontology.txt these define which ontology identifier is to be used, and in custEnt.txt these define the custType of the named entities that are recognised. Furthermore, there is a section called [DEFINE], which allows you to define macros that may be used in the rest of the file.

Further details of these files are given in the default neTerms.txt file.

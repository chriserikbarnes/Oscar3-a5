Name Recognition

Name recognition is a complex process, involving a variety of stages and modules - and a variety of options and forms of data. There are essentially two strategies for this that you can choose between: using a MEMM (Maximum Entropy Markov Model - new in Oscar3 version alpha3 - the default) or using pattern matching (the old method). There are a few shared steps in both processes. In overview:

  1. Finding places for name recognition
  2. Tokenisation
  3. Use of the MEMM or pattern matching
  4. Identification of datasections

Finding places for name recognition

To do this, the SciXML is examined, and all of the paragraphs, section headings, figure/table/etc. captions etc. and the paper title are identified as places for name recognition. This simple process is not affected by options or data.

Tokenisation

This takes each paragraph (etc.) in turn, and splits it into tokens (words, punctuation characters, etc.). This uses the resource file uk/ac/cam/ch/wwmm/oscar3/terms/resources/noSplitPrefixes.txt and the afterHyphen wordset from the model file you are using. A token-level regex is also used.

Use of the MEMM or pattern matching

See the MEMM or pattern matching documentation.

Identification of datasections

Datasections are identified in several manners. If a paragraph consists of 10% or fewer tokens that match the regex [a-z]*[aeiou][a-z]*, then the entire paragraph is marked up as a datasection. If a paragraph belongs in a DIV with the HEADER of "X-Ray data" or similar then the entire paragraph is marked up as a datasection. Otherwise, several heuristics are used to guess if the paragraph contains a data section, and where it starts. This process generates named entities of type "dataSection", which get converted into dataSection elements in the inline XML.

Reload Modules

This allows you to reload various modules from Oscar3

  • All modules
  • OPSIN - dependent upon the contents of uk/ac/cam/ch/wwmm/opsin/resources/
  • Properties file
  • ChemNameDict - NB, also reloads the DFA Name recogniser and the n-grams
  • Named Entity Types and Subtypes - dependent upon uk/ac/cam/ch/wwmm/oscar3/misc/resources/neTypes.xml
  • The PubChem SQL Server
  • The hyphen tokeniser - this depends on the model file and also uk/ac/cam/ch/wwmm/oscar3/terms/resources/noSplitPrefixes.txt
  • Reload the n-gram classifier - see the help file for details and dependencies.
  • Reload the Data Parser - see the help file for details and dependencies.
  • Reload the TLRs - see the help file for details and dependencies.
  • Reload term sets - i.e. the contents of uk/ac/cam/ch/wwmm/oscar3/terms/resources/. This also causes the DFA name recogniser and the ngrams to reload.
  • Reload the DFA name recogniser - see the help file for details and dependencies.
  • Reload the name resolution system
  • Reload the structure diagram generator - dependent upon the contents of uk/ac/cam/ch/wwmm/ptclib/cdk/resources/
  • Reload the model file

New PubChem Mirror for Oscar3

This file explains the (new as of alpha5) method of storing PubChem records, to assist with name resolution. Rather than using SQL bases, this new method uses simple on-disk data structures to store the data.

Initialisation

First, make sure you have set up a workspace for Oscar3. Next, run Oscar3 from the command line with the FetchPubChem command. And wait for a few hours, while Oscar3 downloads the relevant files from PubChem (several gigabytes of them - it may be a good idea to inform your systems administrator or other users of your network connection before doing this), analyses them, and builds an index. And that's it! No further action is required.

Alternatively, you can fetch the relevant files from PubChem yourself. You will need all of the .sdf.gz files in ftp://ftp.ncbi.nlm.nih.gov/pubchem/Compound/CURRENT-Full/SDF, and also the files CID-MeSH and CID-Synonym.gz from ftp://ftp.ncbi.nlm.nih.gov/pubchem/Compound/Extras. These should be placed in the directory specified by the pcdir property in the Oscar3 properties file. If this has not already been set, or has been set to none, you will need to set it yourself. Having fetched the relevant files, run Oscar3 with the BuildPubChem command.
